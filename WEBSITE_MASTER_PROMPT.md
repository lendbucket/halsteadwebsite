## 3. NON NEGOTIABLE RULES

These are absolute. Violating any of them means the work is rejected.

1. No em dashes and no en dashes anywhere in any copy, ever. Use periods, commas, colons, and parentheses. Dashes are the single clearest tell of AI writing and they will get the whole build rejected.
2. No emojis in any website content.
3. All writing reads as a sharp, confident human expert. No AI cliches. Banned phrases include unlock, elevate, delve, navigate the landscape, in today's fast paced world, look no further, in conclusion, whether you are X or Y, the phrase that says something is a game changer, and any filler that adds words without meaning.
4. Vary sentence length and rhythm. Use concrete specifics and real numbers. Sound like an operator who has done this hundreds of times.
5. Never fabricate testimonials, client logos, awards, press mentions, certifications, or results the company has not earned. If there is no social proof yet, build authority through education and transparency instead.
6. SEO metadata and titles are the top priority on every page.
7. Every page must have a keyword rich H1 in a hero that also captures the visitor psychologically.
8. Preserve all existing content and SEO during any audit or upgrade. Never delete pages, routes, metadata, or schema unless explicitly told to.
9. Mobile must feel native, with zero horizontal scroll.
10. Every build must pass a clean production build before it ships.

## 4. TECH STACK AND ENVIRONMENT

Use this stack unless told otherwise:

- Next.js 15 App Router with TypeScript and React 19.
- Tailwind CSS version 3.4 with a config file. Do not use Tailwind version 4.
- Fonts loaded through next/font/google as CSS variables.
- Resend for all form email delivery.
- Vercel for hosting and deploys, connected to GitHub for automatic deployment on push.
- Cloudflare for DNS.
- Pin Next.js to a current patched release, at minimum 15.5.19, to avoid known security advisories in earlier 15.x versions.

Folder and code standards:

- Use the src directory structure with src/app for routes, src/components for shared components, and src/data for structured content like city lists and FAQ data.
- Keep marketing sites statically generated. Do not client render content that needs to be indexed.
- Keep a clean component architecture so pages compose from reusable parts.
- Put site configuration, navigation, contact details, and any pricing or package data in a single config file so it is edited in one place.
- Do not build authentication or a database into a marketing site. Any portal or app belongs in a separate repository on a subdomain so the marketing site stays fast and clean.

Environment variables:

- RESEND_API_KEY for the email service.
- LEAD_FROM_EMAIL for the branded sender, with a fallback to the Resend sandbox sender until the domain is verified.

## 5. SEO MASTER SECTION

This is the core discipline. Treat every item as required.

### 5.1 Keyword strategy

- Identify the primary high intent keywords the business must own. These are the phrases a ready buyer types.
- Identify secondary keywords that support the primary set.
- Identify long tail and question keywords for blog posts and resource pages.
- Map exactly one primary keyword to each core page and each industry or service page.
- Map one long tail question to each blog post.
- Use the hub and spoke model. Broad pillar pages link down to specific pages, and specific pages link back up to the pillar.
- Research and expand the keyword set during the build rather than relying only on what is listed here.

### 5.2 On page SEO, every page

- Unique title tag under 60 characters that includes the primary keyword and the brand.
- Unique meta description around 150 to 160 characters that includes the keyword and a reason to click.
- Canonical URL set and consistent with the primary domain.
- OpenGraph and Twitter card tags.
- Exactly one H1 per page, in the hero, carrying the primary keyword.
- Logical heading hierarchy with the keyword and its variations in at least one H2.
- The primary keyword appears in the first 100 words.
- Clean, readable, keyword based URL slug.
- Descriptive internal links with meaningful anchor text.

### 5.3 Technical SEO

- Dynamic sitemap at /sitemap.xml generated from all routes including blog, industry, location, and resource pages.
- robots file that allows crawling and points to the sitemap.
- Resolve the www versus non www question once. Pick a primary, redirect the other, and make the canonical strategy and the metadataBase match the primary exactly. Mismatched canonicals split ranking authority.
- JSON LD structured data by page type: Organization and WebSite on the layout, Service on service pages, BlogPosting on every article with author and datePublished, FAQPage on FAQ content, BreadcrumbList site wide, HowTo on process pages, LocalBusiness on location pages, DefinedTermSet on a glossary.
- A custom 404 page.
- An RSS feed for the blog.
- Fast Core Web Vitals. Minimal client JavaScript, no layout shift, fast Largest Contentful Paint.

### 5.4 Image SEO

- Every image carries descriptive, keyword aware alt text that names the subject and includes the brand name where natural, for example a description that includes the company name and what the image shows.
- Every locally hosted image uses a brand based filename, for example brandname-hero.jpg and brandname-logo.png, so the filename itself signals relevance.
- Remote images from free sources still get full descriptive alt text.

### 5.5 Lazy loading done correctly

- Lazy load all images below the fold.
- Never lazy load the hero image or any Largest Contentful Paint image. Mark it priority so it loads immediately. Lazy loading the hero is a common mistake that wrecks LCP and rankings.
- Use the framework image component with correct sizes and explicit width and height to prevent layout shift.

### 5.6 Local and hyper local SEO

- Build a location hub plus individual city pages for the services offered, targeting terms like service in city and service company city state.
- Start with the major metros, roughly the top 50 to 100 cities, rather than every town, so each page can carry real content.
- Each city page must have genuinely useful, locally relevant content. Local market context, local industry notes, and a real reason the service matters there. Vary the content meaningfully.
- Never ship thin doorway pages that only swap the city name. Google suppresses those and it can hurt the whole domain.
- Add LocalBusiness and BreadcrumbList schema with service area on every location page.
- Internally link city pages to each other, to the relevant service pages, and to the blog.
- Build the city set from a structured data file so more cities can be added over time.
- Pair the site with a Google Business Profile and consistent name, address, and phone where applicable.

### 5.7 Branded search dominance, sitelinks, and the knowledge panel

- The owner wants a search for the brand name to surface many pages, with sitelinks under the main result and ideally a knowledge panel on the side.
- These cannot be switched on. They are earned. Support them by: a clear navigation with descriptive labels, strong internal linking, consistent use of the brand name in titles, headings, and copy across the whole site, and complete Organization schema.
- Add sameAs links in the Organization schema pointing to the company's real social profiles to support entity recognition and a future knowledge panel.
- Submit the sitemap in Google Search Console so the pages get indexed. Google cannot rank what it has not crawled.

### 5.8 Content and authority

- Education is the strategy. Teaching builds authority, earns trust, and is what makes a visitor feel they need the expert.
- Build genuine depth: pillar guides, a blog of substantial posts, a glossary, and resource pages.
- Write for E E A T. Demonstrate real experience, expertise, authority, and trust through specific, accurate, useful content.
- Every educational page opens at least one curiosity loop that points the reader deeper into the site.

## 6. DESIGN SYSTEM, FORTUNE 500 NOT SAAS

The look must read as an established enterprise brand, never a generic SaaS template.

What makes a site feel like SaaS, and must be avoided: all sans serif type, gradient pill buttons, rounded card grids repeated everywhere, bright playful color, illustration and icon soup, and identical section blocks stacked down the page.

What makes a site feel Fortune 500, and must be applied:

### 6.1 Typography

- Lead with a refined serif display for headlines, at large sizes, with tight letter spacing. This single choice moves a site away from SaaS faster than anything else.
- Pair the serif with a clean, professional sans for body and UI.
- Build a deliberate type scale and use it consistently. Use fluid sizing with clamp so headlines scale cleanly from small phones to large desktops.
- Use small, letter spaced uppercase labels as eyebrows above headlines.

### 6.2 Color

- Use the brand palette with restraint. The accent color is a precise tool for primary actions and highlights, not a background to splash everywhere.
- Use a warm, premium foundation. Let dark sections appear sparingly for gravitas.

### 6.3 Layout

- Generous whitespace and a strong editorial grid. Let content breathe.
- Vary section layouts. Alternate image led, text led, full bleed, and asymmetric sections. Repetitive identical sections are what make a site feel machine generated.
- Use thin hairline rules to divide sections rather than stacking colored cards.
- One clear focal point per section.

### 6.4 Photography

- Use real photography, not illustration or icon grids.
- Pull only free license images from reputable free sources, and make sure every image genuinely matches the content beside it.
- Apply a consistent warm treatment or subtle duotone so images feel intentional and on brand rather than random stock.

### 6.5 Components and details

- Confident, squared or slightly radiused buttons. No gradient pills.
- Refined cards with hairline borders and a subtle lift on hover.
- Editorial touches: pull quotes, large stat blocks with serif numerals, precise alignment.

### 6.6 Logo handling

- Process logo files to have true transparent backgrounds at the pixel level. Do not rely on CSS blend tricks that break on some backgrounds.
- Use an adaptive logo, the dark version on light backgrounds and the light version on dark backgrounds.
- Size the logo with intent. Never let it look tiny, cramped, or boxed.
- Homepage header pattern when requested: a transparent header over the hero with the navigation accessible, a large centered brand logo in the hero, and the header transitioning to a solid background on scroll so it stays readable. Inner pages keep the standard solid header.

### 6.7 Centering discipline

- Center buttons within their containers.
- Center the content inside ribbon, band, and strip sections.
- Audit every page for alignment and fix anything that should be centered and is not.

## 7. BEHAVIORAL PSYCHOLOGY AND CONVERSION

Architect the whole site as a deliberate psychological and educational journey from visitor to lead.

### 7.1 Reduce cognitive overload

- Keep all the information, but reveal it in layers. Do not dump everything at once.
- Progressive disclosure: use expandable sections, tabs, and read more reveals for dense material.
- Chunking: break long content into scannable groups of three to five, and add a short key takeaways summary at the top of long pages so a skimmer gets value in ten seconds.
- Reduce competing choices in any single view. One primary action per section.
- Use whitespace and pacing so the reader's eyes never get tired.
- On long pages add a sticky table of contents and a reading progress bar so the page feels navigable and finite.

### 7.2 Hold attention and pull deeper

- Open loops, the Zeigarnik effect. End sections with a curiosity gap that points to the next page, then link to it. Build a trail of curiosity through the site.
- Continue learning modules. End every page with a related content block of three hand picked next steps.
- A guided learning path. Build a start here sequence of numbered steps through the key pages. This uses the goal gradient effect, since people finish what they start, and it routes them toward the lead actions.
- Inline interactivity. Surface tools like assessments and calculators throughout the site, not only on their own pages.

### 7.3 The conversion journey

Every page plays one clear role in this sequence:

1. Hook. The hero grabs identity and aspiration in the first three seconds.
2. Stakes. Agitate the cost of doing it wrong so the visitor feels they need an expert.
3. Authority. Process, education, and proof of expertise.
4. Micro commitment. A low friction first yes, usually an assessment, a calculator, or a checklist.
5. Education builds trust. Resources and guides earn the lead through reciprocity.
6. Conversion. The booking and the lead form.

### 7.4 Calls to action

- Use specific, value driven CTAs, never generic repeated phrases like get started or learn more.
- Make CTAs educational where possible, for example a free score, a free checklist, or a free assessment.
- Isolate the single primary CTA in any view so the eye goes straight to it.
- Place CTAs at natural decision points, right after a section delivers real value.
- Add short reassurance microcopy under important CTAs to reduce friction.
- Add a context aware sticky CTA that adapts to the page.

### 7.5 Lead capture

- Use a multi step form with a progress indicator. Multi step forms feel lighter, convert higher, and use the foot in the door effect.
- Capture the email on the first step so a partial completion is still a usable lead.
- Gather all the qualifying data the business needs to score the lead.
- Gate high value tools like calculators and assessments behind an email so they become lead sources.
- Send submissions through Resend to the lead email with loading, success, and error states.

### 7.6 The Hormozi and Gary Vee voice

- Lead with hooks. Give enormous value for free first, then ask. Jab, jab, jab, right hook.
- Stack value so the price feels small against what is received.
- Use numbers and return on investment framing, because business owners respond to numbers.
- Make the next step feel like an obvious decision.

## 8. MOBILE NATIVE STANDARDS

The site must feel like a native app on a phone.

- Zero horizontal scroll. Audit every page at 320, 375, 390, and 430 pixel widths.
- Add a global overflow guard, and hunt down every overflow cause: fixed widths, oversized images, grids that do not stack, negative margins, full viewport width elements, wide tables, and long unbroken strings. Wide content scrolls inside its own container, never the page.
- Touch targets at least 44 by 44 pixels with visible active and tap feedback.
- Do not rely on hover only interactions on touch devices.
- A full screen, app style mobile menu that slides over the page with large tappable rows and accordion sections, not a small dropdown list.
- Use the dynamic viewport height unit for full height elements so mobile browser chrome does not break layouts.
- Respect safe area insets for notched phones.
- Enable momentum scrolling and confirm the page scrolls only vertically.
- Fluid typography with clamp so headlines scale down cleanly.

## 9. INTERACTIVITY STANDARDS

Add tasteful, premium interactivity across the whole site. Never bouncy or playful.

- Reveal on scroll for sections and images using an intersection observer with smooth easing and short distances.
- Subtle lift and shadow on interactive cards.
- Animated underlines on text links.
- Count up animation on stat numbers when they enter the viewport.
- Subtle, intentional motion on hero elements, including a gentle entrance on the hero logo where used.
- Expandable and tabbed elements where they aid understanding.
- Respect prefers reduced motion everywhere and disable animation for those users.

## 11. LEGAL AND COMPLIANCE

- Add a privacy policy page and link it in the footer. Tailor it to the data the site actually collects through its forms, plus automatically collected technical data, third party service providers, cookies, retention, user rights including California and international rights, security, children, third party links, changes, and a contact. Note that a lawyer should review it for the jurisdiction.
- Add a terms page.
- Never publish testimonials, client logos, or results the company has not earned.
- For any regulated offering, add the proper disclaimers and route the sale through the legally required process. Do not make claims a disclaimer cannot cure.
- If the business sells franchises: selling franchises is governed by the FTC Franchise Rule and state laws. Franchises are offered only by a Franchise Disclosure Document delivered at least 14 days before signing or payment. Any earnings claim is a Financial Performance Representation that is lawful only if it is in Item 19 of the FDD with a reasonable basis. A disclaimer does not make an unsupported earnings claim legal. Display the franchise fee with the standard offering disclaimer, do not publish a personal income calculator without an Item 19, and recommend an attorney review before selling.
- Add the relevant industry disclaimers for the specific business and recommend professional review.

## 13. THE BUILD AND DEPLOY PLAYBOOK

Follow this exact flow, including the hard won gotchas.

### 13.1 Repository and local build

- Create or use the GitHub repo under the owner's account.
- Scaffold or work in the project, keep the build passing with a clean production build at every milestone.
- Process logo assets to true transparency before using them.

### 13.2 Push to GitHub

- Configure git identity and authenticate GitHub once with the gh CLI, then run the gh git credential setup so pushes work without prompts. On Windows over SSH the standard credential store can fail, and the gh credential helper is the reliable fix.
- Commit with clear messages and push to main.

### 13.3 Vercel deploy, and the framework gotcha

- The repo connects to Vercel and deploys on push.
- Critical: confirm the Vercel project Framework Preset is set to Next.js. If a project is created before code exists, Vercel can leave the framework unset, which builds successfully but returns 404 on every route. If you see a ready deployment that 404s, this is almost always the cause. Set the preset to Next.js and redeploy.
- Set the environment variables, then redeploy, because variables only take effect on deployments created after they are added.

### 13.4 Domain and Cloudflare, and the caching gotcha

- Add the domain in Vercel, set the primary, and redirect the other of www and non www to it.
- In Cloudflare DNS, add the records Vercel specifies. Set the records that point to Vercel to DNS only, the grey cloud, not the orange proxied cloud. The orange proxy breaks Vercel SSL and, just as important, caches your HTML so new deploys do not appear to visitors or to Google. If changes do not show after a deploy, purge the Cloudflare cache, and switch the records to grey cloud so it stops caching going forward.
- Set the SSL mode to Full if any record stays proxied.

### 13.5 Email deliverability with Resend

- Verify the domain in Resend and add the SPF, DKIM, and DMARC records it generates into Cloudflare DNS exactly. Any CNAME record, including DKIM, must be grey cloud DNS only.
- Set the branded sender once the domain is verified, then redeploy.
- Submit a real test through the form and confirm a lead arrives, checking spam the first time.

### 13.6 Google Search Console

- Add a domain property and verify it with a DNS TXT record in Cloudflare. A domain property covers www and non www together.
- Submit the sitemap so Google indexes the pages.

### 13.7 Long agentic runs over a remote machine

- When driving a build remotely over SSH, the client machine must stay awake or the connection drops and kills the run. On a Mac client, keep a caffeinate session running. Prevent the host from sleeping and set server keep alive so the link holds.
- Work survives to disk as the agent writes, and the session can be resumed from the project directory, so a drop interrupts the current step but does not undo finished files.

## 14. THE AUDIT PROTOCOL FOR EXISTING SITES

When auditing an existing site, do not just glance at the homepage. Crawl and score the whole site against every standard, deliver a written findings report grouped by severity, then execute the fixes in priority order. Preserve all existing content and SEO while upgrading.

Run through this checklist page by page and pillar by pillar:

### 14.1 Technical SEO audit

- [ ] Every page has a unique title under 60 characters with the primary keyword.
- [ ] Every page has a unique meta description around 150 to 160 characters.
- [ ] Canonical tags are present and consistent with the primary domain.
- [ ] www versus non www is resolved with a redirect and matching canonicals.
- [ ] Sitemap exists, is current, and includes every page.
- [ ] robots allows crawling and references the sitemap.
- [ ] Structured data is present and valid for each page type.
- [ ] No broken links and no orphan pages.
- [ ] Clean keyword based URL slugs.
- [ ] Custom 404 exists.

### 14.2 On page and content audit

- [ ] Exactly one H1 per page, in the hero, with the primary keyword.
- [ ] Heading hierarchy is logical and keyword aware.
- [ ] Primary keyword appears in the first 100 words.
- [ ] Content has real depth and demonstrates experience and expertise.
- [ ] Internal links use descriptive anchor text and follow hub and spoke.
- [ ] No thin or duplicate pages, especially across location pages.
- [ ] Zero em dashes and en dashes. Zero emojis. No AI cliches.
- [ ] No fabricated testimonials, logos, or results.

### 14.3 Image and performance audit

- [ ] Every image has descriptive, brand aware alt text.
- [ ] Local images use brand based filenames.
- [ ] Below the fold images lazy load, the hero does not.
- [ ] Images use the framework image component with sizes and dimensions.
- [ ] Core Web Vitals are healthy, LCP fast, no layout shift, minimal client JS.

### 14.4 Design audit

- [ ] Reads as Fortune 500, not SaaS.
- [ ] Serif display headlines paired with a clean body font.
- [ ] Palette used with restraint, accent reserved for actions.
- [ ] Varied section layouts, generous whitespace, hairline dividers.
- [ ] Real photography with consistent treatment, matched to content.
- [ ] Logo is transparent, adaptive, and properly sized.
- [ ] Buttons and ribbon content are centered.

### 14.5 Psychology and conversion audit

- [ ] No page overwhelms at first glance.
- [ ] Progressive disclosure on dense content.
- [ ] Open loops point deeper into the site.
- [ ] A guided path exists and routes to the lead action.
- [ ] Every page has one clear, specific, value driven next step.
- [ ] CTAs are isolated, specific, and placed after value.
- [ ] Lead form is multi step, captures email first, and gathers full qualifying data.

### 14.6 Mobile audit

- [ ] Zero horizontal scroll at 320, 375, 390, and 430 pixels.
- [ ] Touch targets at least 44 pixels with tap feedback.
- [ ] Full screen app style mobile menu.
- [ ] Safe area insets respected.
- [ ] Layout, spacing, and grids collapse cleanly on small screens.

### 14.7 Legal and trust audit

- [ ] Privacy policy and terms exist and are linked.
- [ ] Required industry disclaimers are present and well designed.
- [ ] Any regulated claims are compliant.

Deliver the findings as a report, then fix everything, then run the self audit in Section 15, then deploy.

## 16. APPENDIX A: HALSTEAD SECURITY MODULE

Use this when the company is Halstead Security.

### A.1 Positioning

Halstead Security is a security services company. The audience is businesses, property managers, event organizers, and individuals who need protection and peace of mind. The emotional drivers are safety, trust, reliability, and the fear of what happens without protection. Sell certainty and professionalism.

### A.2 Keyword targets

- Primary: security guard services, security company, private security, security services near me.
- Service specific: event security, executive protection, residential security, commercial security, construction site security, retail loss prevention, mobile patrol, fire watch, concierge security, alarm response.
- Local: security guard services in city, security company city state, event security city. Local SEO is the biggest lever in this industry, so build the city pages aggressively and substantively.

### A.3 Site architecture additions

- A service page for each security service offered, each targeting its own keyword.
- An industries served set: corporate, retail, residential, construction, events, healthcare, education, hospitality, warehouse and logistics.
- A heavy location hub with city pages, since security buyers search locally.
- A why Halstead page that builds trust through professionalism, screening, training, licensing, response times, and technology.
- A request a security assessment or request a quote flow as the primary lead capture.

### A.4 Trust and authority for security

- Trust is the entire sale. Emphasize vetting, training, licensing, insurance, supervision, technology, and reliability. Use real, true specifics only.
- Do not promise outcomes you cannot guarantee. Avoid absolute safety guarantees. Frame the value as professional protection and risk reduction.
- If guards or the company hold state licenses, reference them accurately. Security guard licensing varies by state, so keep claims truthful and current.

### A.5 Imagery

- Real photography of professional security personnel, properties, events, and command operations, with the warm enterprise treatment. Avoid anything that looks like a stock cliche or implies force or intimidation.

### A.6 Compliance

- Add the privacy policy and terms.
- Keep all licensing, certification, and capability claims accurate and verifiable.
- Recommend legal review of any guarantees or service claims.
